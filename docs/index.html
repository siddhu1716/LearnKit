v0.1 Architecture

Two paths. One memory store.

Pick the decorator that fits how your agent runs — both write to the same persistent backend and share the same quality gates.

Path A · @lk.learn

Model path

For any LLM agent. Distilled skills and failures are injected as text context — your model gets richer prompts and compounds quality over time. No tool instrumentation needed.

@lk.learn(domain="legal") def agent(task, _learnkit_context=""): # distilled context injected on every call return llm.run(task, system=_learnkit_context)
  • ✦ Works with any framework or model
  • ✦ Quality lift on novel sibling tasks
  • ✦ Zero tool-instrumentation needed
Path B · @lk.agent_learn · New in v0.1

Tool-agent path

For agents that call tools. Captures every tool sequence, replays proven procedures on exact re-encounters, and guides similar tasks — slashing LLM overhead by up to 66% across rounds.

@lk.agent_learn(domain="pipeline") def tool_agent(task, _learnkit_tools=None): rows = _learnkit_tools.record("query", args, result) # procedure captured ↑ replayed next run return rows
  • ✦ Tool-call capture & procedure replay
  • ✦ Up to 66% fewer LLM calls (multi-round)
  • ✦ Zero-LLM cost on exact re-encounters
